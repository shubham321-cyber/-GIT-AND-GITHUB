Git and GitHub

Git is a Version Control System that tracks changes in your code. GitHub is a Cloud Platform where you store your code (like Google Drive for developers).
   


Command,Simple English Meaning

git init,"Starts a new Git ""Brain"" inside your folder (creates .git)."
ls -a,Shows hidden files like .git. Never edit files inside the .git folder!
git status,"Shows the ""Health Report"" (which files are new or changed)."
git add <file>,Moves a file from the folder to the Staging Area (Ready to save).
"git commit -m ""msg""",Permanently saves your changes in your Local PC with a message.
git log,Shows the history of all your previous commits (saves).
 git restore  testing,txt: restore the file which we remove
 
The Daily Workflow (3 Steps)
Once your folder is connected to GitHub, you only need to remember these three commands to save your work every day:

Step 1: git add .
What it does: It tells Git, "Hey, look at all the changes I made in this folder and get them ready to be saved."

Analogy: Putting your items into a box before shipping them.

Step 2: git commit -m "Your Message"
What it does: It creates the Save Point. The message (e.g., "Added Linux notes for Day 5") helps you remember what was changed.

Analogy: Putting a label on that box so you know what is inside.

Step 3: git push
What it does: It sends your labeled box to the GitHub servers.

Analogy: Handing the box to the delivery truck to take it to the warehouse (GitHub).
